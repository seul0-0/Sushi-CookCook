## Commit Convention
```text
Feat:     새로운 기능 추가
Fix:      버그 수정 또는 typo
docs:     문서작업 (Documentation)
Refactor: 리팩토링
Design:   CSS 등 UI 디자인 변경
Comment:  필요한 주석 추가 및 변경
Style:    코드 포맷팅, 세미콜론 누락 (코드 변경 없음)
Test:     테스트 코드 추가/수정/삭제 (비즈니스 로직 영향 없음)
Chore:    빌드 스크립트, 에셋, 패키지 등 기타 변경
Init:     프로젝트 초기 생성
Rename:   파일/폴더명 수정 또는 이동
Remove:   파일 삭제
```
